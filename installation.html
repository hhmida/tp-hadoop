    Installation sous Windows

    Bien que l'installation sous Windows ne soit pas recommandée, elle est possible. En effet, Un Wiki Apache introduit une méthode d'installation de Hadoop 2 sous Windows. Un autre guide, par Divya Singh, montre les étapes d'installation de Hadoop 3.1.0 sous Windows 10.

    Installer les Prérequis¶

    Hadoop est développé en Java, une version du jdk est donc nécessaire à son exécution. Avec la version 3, c'est la version 8 ou plus.

    Mettre à jour la liste des sources

    $ sudo apt-get update

    Installer le JDK 8

    $ sudo apt-get install openjdk-8-jdk

    Configurer SSH pour une connexion sans mot de passe

    Générer une paire de clé privée/publique.

    $ ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa

    Autoriser l'accès avec la nouvelle clé 

    $ cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys
$ chmod 0600 ~/.ssh/authorized_keys

    Installer Hadoop¶

    Procédure d'installation¶

    Télécharger les binaires de la version 3.2.1

    Le fichier à installer est sous la forme tar qui peut être téléchargé avec wget :

    $ wget https://downloads.apache.org/hadoop/common/stable/hadoop-3.2.1.tar.gz

    Si le fichier est présent dans la machine Windows, utiliser WinSCP pour le copier vers la machine VirtualBox :

    Extraire les fichiers

    $ tar -xzvf hadoop-3.2.1.tar.gz
$ sudo mv hadoop-3.2.1 /usr/local/hadoop
    Configurer la variable JAVA_HOME

    Ajouter le chemin de java dans le fichier /usr/local/hadoop/etc/hadoop/hadoop-env.sh :

    $ cd /usr/local/hadoop
$ echo "export JAVA_HOME=/usr/lib/jvm/java-8-openjdk-amd64" >> etc/hadoop/hadoop-env.sh

    Vérifier l'exécution de Hadoop

    Il est plus pratique d'ajouter une variable d'environnement HADOOP_HOME qui pointe vers le dossier d'installation de Hadoop et d'ajouter les dossiers $HADOOP_HOME/bin et $HADOOP_HOME/sbin à la variable PATH pour simplifier le démarrage des différents services. Pour cela, il faut modiffier le fichier .bashrc ainsi :

    • Ouvrir le fichier .bashrc :
    $ nano ~/.bashrc
    • Ajouter les lignes :
    export HADOOP_HOME=/usr/local/hadoop
export PATH=$PATH:$HADOOP_HOME/sbin:$HADOOP_HOME/bin

    • Appliquer les changements
    $ source ~/.bashrc


    $ hadoop version

    Configuration de Hadoop¶

    Les fichiers de configuration de Hadoop sont au format XML et sont localisés dans le dossier etc/hadoop dont principalement :

    • core-site.xml : Paramètres généraux.
    • hdfs-site.xml : Paramètres HDFS.
    • mapred-site.xml : Paramètres MapReduce.
    • yarn-site.xml : Paramètres YARN.

    La liste des paramètres et leurs valeurs par défaut est disponible sur le site officiel dans les fichiers :

    • core-default.xml pour le fichier core-site.xml.
    • hdfs-default.xml pour le fichier hdfs-site.xml.
    • mapred-default.xml pour le fichier mapred-site.xml.
    • yarn-default.xml pour le fichier yarn-site.xml.

    Configuration de HDFS¶

    core-site.xml¶

    Ce fichier contient le paramétrage général du système de fichiers (HDFS, Azure, Amazon, ...), des tampons, des délais et intervalles des E/S, etc.

    Afin de définir le système de fichier par défaut à HDFS qui opère sur le port 9000, modifier le fichier core-site.xml ainsi :

    <configuration>
    <property>
        <name>fs.defaultFS</name>
        <value>hdfs://localhost:9000</value>
    </property>
</configuration>

    hdfs-site.xml¶

    C'est le fichier de configuration de HDFS. Dans ce qui suit nous définissons le taux de réplication dfs.replication des blocs HDFS à 1 au lieu de 3 par défaut ainsi que le dossier pour enregistrer la structure HDFS (fsimage) ainsi le chemin pour le stockage des blocs.

    <configuration>
    <property>
        <name>dfs.replication</name>
        <value>1</value>
    </property>
    <property>
        <name>dfs.namenode.name.dir</name>
        <value>file:/usr/local/hadoop/dfs/namenode</value>
    </property>
    <property>
        <name>dfs.datanode.data.dir</name>
        <value>file:/usr/local/hadoop/dfs/datanode</value>
    </property>
    <property>
        <name>dfs.permissions.enabled</name>
        <value>false</value>
    </property>
</configuration>

    La propriété dfs.permissions.enabled est désactivée pour permettre l'écriture sur le système HDFS à partir de l'interface web lors de l'accès à partir de la machine Windows.

    Maintenant, il faut créer les dossiers spécifiés dans le fichier hdfs-site.xml et changer le propriétaire pour avoir les droits de lecture et écriture.

    $ cd
$ mkdir -p /usr/local/hadoop/hdfs/namenode
$ mkdir -p /usr/local/hadoop/hdfs/datanode
$ sudo chown iset:iset -R /usr/local/hadoop

    Service HDFS¶

    1. Formatage
    $ hdfs namenode -format
    1. Dénarrage de HDFS
    $ start-dfs.sh

    1. Créer l'arboresence de l'utilisateur principal.
    $ hadoop fs mkdir -p /user/iset
    1. Vous pouvez accéder depuis la machine hôte à l'interface : http://192.168.100.10:9870

    Il est possible d'explorer le contenu du système de fichiers à partir du menu Utilities puis Browse the file system.

    MapReduce et YARN¶

    mapred-site.xml

        <configuration>
        <property>
            <name>mapreduce.framework.name</name>
            <value>yarn</value>
        </property>
        <property>
            <name>yarn.app.mapreduce.am.env</name>
            <value>HADOOP_MAPRED_HOME=${HADOOP_HOME}</value>
        </property>
        <property>
            <name>mapreduce.map.env</name>
            <value>HADOOP_MAPRED_HOME=${HADOOP_HOME}</value>
        </property>
        <property>
            <name>mapreduce.reduce.env</name>
            <value>HADOOP_MAPRED_HOME=${HADOOP_HOME}</value>
        </property>
    </configuration>

    yarn-site.xml

        <configuration>
        <property>
            <name>yarn.nodemanager.aux-services</name>
            <value>mapreduce_shuffle</value>
        </property>
        <property>
            <name>yarn.nodemanager.auxservices.mapreduce.shuffle.class</name>
            <value>org.apache.hadoop.mapred.ShuffleHandler</value>
        </property>
    </configuration>

    Démarrage du service

    Il suffit d'exécuter le script start-yarn.sh :

    $ start-yarn.sh

    L'interface du Resource Manager est accessible depuis : http://192.168.100.10:8088

    Démarrage et arrêt de Hadoop
    $ start-dfs.sh
$ start-yarn.sh

    $ stop-dfs.sh
$ stop-yarn.sh
